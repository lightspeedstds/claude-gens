What is this?

These are Claude Code agents — specialised AI assistants that live as .md files. You invoke them by name inside Claude Code (e.g. @gmail-inbox-triage triage my inbox) and they use real tools to get things done: send emails, edit spreadsheets, clean your drive, debug webpages, and more.

They work through MCPs (Model Context Protocol servers) — connectors that give Claude access to external services like Gmail, Google Drive, and Chrome DevTools.

ℹ️

You need Claude Code (the CLI or VSCode extension) — not just the claude.ai website. Agents don't work in the browser chat.

Prerequisites

Make sure these are installed before running setup:

  • macOS or Linux — most agents are cross-platform. Note: ram-optimizer uses macOS-only tools (vm_stat, osascript) and will not run on Linux
  • Node.js 18+ — needed to run MCP servers via npx. Download from nodejs.org
  • Python 3.7+ — needed by lead-extractor and text-sanitizer. Setup auto-detects python3 or python
  • Claude Code — the CLI or VSCode extension. Get it from claude.ai/code
  • Google Chrome — only needed for Chrome DevTools agents. Install via brew install --cask google-chrome (macOS) or from google.com/chrome
  • A Google account — for Gmail, Calendar, Drive, and Workspace agents
⚠️

Check your Node version: node --version — must be v18 or higher.

2

Download & Install

Clone or download the project, then run the setup script. It patches all agent files with your local paths and registers MCPs with Claude Code automatically.

Option A — git clone

git clone https://github.com/your-repo/claude-agents ~/claude-agents cd ~/claude-agents bash setup.sh

Option B — download ZIP

Download and unzip anywhere, then open Terminal in that folder and run:

bash setup.sh

Open in Claude Code

After setup, open Claude Code with this folder as the workspace:

claude ~/claude-agents
✅

The setup script replaces all hardcoded paths, creates the _shared/ directories, and registers Chrome DevTools automatically. Google MCPs require the extra steps below.

Google OAuth Setup

Gmail, Calendar, Drive, and Workspace agents need a Google Cloud OAuth2 credential. You only do this once — the same file works for all of them.

ℹ️

You're creating a private "app" in Google Cloud that has permission to access your own Google account. Nothing is shared publicly.

1 — Create a Google Cloud project

Go to console.cloud.google.com → click the project dropdown → New Project → name it anything (e.g. ClaudeAgents) → Create.

2 — Enable APIs

Go to APIs & Services → Library and enable these one by one:

  • Gmail API
  • Google Calendar API
  • Google Drive API
  • Google Docs API
  • Google Sheets API
  • Google Slides API

Search each one by name and click Enable.

3 — Configure OAuth consent screen

Go to APIs & Services → OAuth consent screen:

  • User type: External → Create
  • App name: anything (e.g. ClaudeAgents)
  • Support email: your Gmail
  • Click through the rest with defaults → Save
  • On the Test users page → Add Users → add your own Gmail address

4 — Create OAuth credentials

Go to APIs & Services → Credentials → Create Credentials → OAuth Client ID:

  • Application type: Desktop app
  • Name: anything
  • Click Create
  • Click Download JSON

5 — Place the credentials file

Rename the downloaded file to gcp-oauth.keys.json and copy it to each service directory:

mkdir -p ~/.gmail-mcp ~/.calendar-mcp ~/.gdrive-mcp ~/.workspace-mcp # Replace ~/Downloads/client_secret_xxx.json with your actual filename cp ~/Downloads/client_secret_xxx.json ~/.gmail-mcp/gcp-oauth.keys.json cp ~/.gmail-mcp/gcp-oauth.keys.json ~/.calendar-mcp/gcp-oauth.keys.json cp ~/.gmail-mcp/gcp-oauth.keys.json ~/.gdrive-mcp/gcp-oauth.keys.json cp ~/.gmail-mcp/gcp-oauth.keys.json ~/.workspace-mcp/gcp-oauth.keys.json

6 — Authenticate each service

Run these one at a time. Each opens a browser window — sign in with your Google account and click Allow.

# Gmail npx @gongrzhe/server-gmail-autoauth-mcp auth # Google Calendar npx -y @gongrzhe/server-calendar-autoauth-mcp auth # Google Drive GDRIVE_OAUTH_PATH=~/.gdrive-mcp/gcp-oauth.keys.json \ GDRIVE_CREDENTIALS_PATH=~/.gdrive-mcp/credentials.json \ npx -y @modelcontextprotocol/server-gdrive auth # Google Workspace (run it once — it will open a browser) CLIENT_ID=your_client_id \ CLIENT_SECRET=your_client_secret \ GWORKSPACE_CREDS_DIR=~/.workspace-mcp \ npx -y @alanse/mcp-server-google-workspace
⚠️

For the Workspace command, replace your_client_id and your_client_secret with the values from your gcp-oauth.keys.json file (the client_id and client_secret fields).

✅

If Google shows "Access blocked" — go back to the OAuth consent screen → Test users → add your email address.

4

Register MCPs with Claude Code

After completing OAuth, run the setup script again — it will detect your credential files and register everything:

bash setup.sh

Or register manually:

claude mcp add -s user chrome-devtools -- npx -y chrome-devtools-mcp@latest claude mcp add -s user gmail-local -- npx @gongrzhe/server-gmail-autoauth-mcp claude mcp add -s user google-calendar-local -- npx -y @gongrzhe/server-calendar-autoauth-mcp claude mcp add -s user google-drive \ -e GDRIVE_OAUTH_PATH=$HOME/.gdrive-mcp/gcp-oauth.keys.json \ -e GDRIVE_CREDENTIALS_PATH=$HOME/.gdrive-mcp/credentials.json \ -- npx -y @modelcontextprotocol/server-gdrive claude mcp add -s user google-workspace \ -e CLIENT_ID=your_client_id \ -e CLIENT_SECRET=your_client_secret \ -e GWORKSPACE_CREDS_DIR=$HOME/.workspace-mcp \ -- npx -y @alanse/mcp-server-google-workspace

Verify everything connected:

claude mcp list

You should see ✓ Connected next to each server. Then restart Claude Code.

How to Use

Open Claude Code in the agents folder and invoke any agent with @:

Examples

# Triage your inbox @gmail-inbox-triage triage my unread emails # Create a Google Sheet @sheets-editor create a budget spreadsheet with columns: Category, Planned, Actual # Clean up your Mac @parallel-orchestrator scan for stale files, duplicates, and corrupted files at the same time # Debug a slow webpage @performance-auditor audit https://example.com # Free up RAM @ram-optimizer my Mac feels slow # Send an email @gmail-drafter email john@example.com saying the meeting is moved to Thursday # Run commands fast @bash-cmd-runner run npm install, then build and lint in parallel # Send event emails to everyone on a list @lead-extractor read FindingBilly.xlsx @text-sanitizer clean all the email bodies @multi-person-gmail-blaster send the update email to everyone

Tips

  • Agents remember their last run — if you run the same scan twice in 24h, they'll skip unchanged files automatically
  • Agents write errors to _shared/problems.md — run @agent-supervisor weekly to health-check everything
  • Add a new MCP? Run @mcp-wrapper-factory to automatically generate focused agents for it
  • Use @parallel-orchestrator for any task with multiple independent parts — it's much faster
✅

All agents use state caching — they skip work they've already done, making repeated runs much faster.

Troubleshooting

MCP not loading after restart

Run claude mcp list in Terminal to see which servers connected. If one shows an error, re-run its auth command.

"Access blocked" during Google OAuth

Your app is in test mode. Go to Google Cloud Console → APIs & Services → OAuth consent screen → Test users and add your Gmail address.

Agent can't find a file

Re-run bash setup.sh — the paths may not have been patched if you moved the folder after initial setup.

Chrome DevTools MCP fails

Make sure Chrome is installed at /Applications/Google Chrome.app. Install via: brew install --cask google-chrome

Workspace token expires

The token auto-refreshes every 45 minutes while the MCP is running. If it expires (e.g. after a long idle), restart Claude Code — the MCP will re-authenticate using the stored refresh token.
